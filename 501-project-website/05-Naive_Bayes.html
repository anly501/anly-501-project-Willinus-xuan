2.Naïve Bayes (NB) in R with Labeled Record Data

My goal is to analyze how to detect cyberviolence automatically. In terms of the variable, I associate each tweets with account id, so the record data within in the account profile may be useful to enhance the model accuracy. Among them, the numbers of followers, the numbers of retweet, and word length and so foth may have some correlation with the ylabel.

As you can see in the Figure 2.1, the confusion matrix of record data indicates that my model behaves very badly. We can see the true positve rate and true negative rate are both very low, only 16%. It indicates that my model actually cannot fully predict and classify whether a context has cyberviolence tendency or not.

Figure 2.1

I also plot a roc curve to further substantiate my argument and evaluate my Naive Bayes model.

As you can see in the Figure 2.2, the auc score is pretty low, only 0.4562. It actually indicates that the probability to predict the label correctly is small. The perfect Roc curve should plot in the upper left side, but this definitely is not shown in my graph.

Figure 2.2

Above all, the experiment shows that the record data has low correlation with the label, which is the cyberviolence property. I might combine the text data, also use some other key indicators of record data to enhance my accuracy.

3.Naïve Bayes (NB) in Python with Labeled Text Data

First of all, consider my variables have discrete property, so the model I use is Gaussian Naive Bayes instead of Multinomial. The text exerts an indispensable role on the overall behavior of the label prediction, so it should have a very high accuracy.

I mainly use three indicators to evaluate the Multinomial Naive Bayes model.

As you can see in the Figure 3.1, the confusion matrix greatly illustrates that my model can predict the true positive label very well. However, it shows that many negative labels also wrongly falls into the false positve categories. which cause the type I error. This kind of error should be mostly avoided in the cyberviolence detection, because it would wrongly erase many non-violence text if using this model.

Figure 3.1

You can also see in the Figure 3.2,this learning curve indicates us that the model is under-fitting, and we require more training size. Since as the training_example increases, the training_score constantly increase and do not halt at a certain point. Besides, The model shows large error in both variance and bias, because the gap between the two curves is large and need to be narrowed.

Figure 3.2

Lastly, in the roc curve,AUC is a metric that utilizes probabilities of the class predictions, we can be more confident in a model that has a higher AUC score than one with a lower score even if they have similar accuracies. In this graph of the model, the Auc score is not pretty high, which means that the model only does a fair job in classification
