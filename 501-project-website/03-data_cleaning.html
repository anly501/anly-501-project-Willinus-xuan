1.Data Cleaning

After collecting data through twitter API and public datasets online, we need to design a pipeline to clean through the data in steps. Therefore, the preprocessed data can be used to explore deeper with EDA and modeling.

There are two hold-out datasets that I used, so we need to both process them into clean ones.

Cleaning method

Since the words contains comma, whitesplace, and other unrelated content between each of them, the regular expression and built-in python package called NLTK are largely used in this research. There are many useful funcitons such as word_tokenize and WordNetLemmatizer that are incredibly effective to process raw data.

For the first one that I collect through tweepy, my experienment designs to analyze tweets in lieu of retweets or reply, so we first need to exclude all tweets that contain 'RT' strings. Besides, the variable that contains geographical information is mainly useless in terms of this experiement, thus deleting it.

Figure 1.1

Also, all the texts require a pipeline to preprocess them into clean format. I build a pipeline function to exclude the stopwords, lemmatize the words, stemmetize the words further, and also leverage the regular regression to cut off the irrelated information.

Figure 1.2

For the dataset collected from public library, it is preprocessed in the same way. However, for the description variable in the first dataset, there are some emotions and pictures left to be preocessed latter for a proper use.
